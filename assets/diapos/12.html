Almacenamiento PostgreSQL

Implementación de persistencia de datos mediante StatefulSets y Persistent Volumes en GKE

PostgreSQL StatefulSet

  • Workload de Base de Datos Puerto 5432 Despliegue utilizando StatefulSet para garantizar identidad de red estable y ordenamiento secuencial de pods.
  • Conexión Centralizada Punto único de verdad para todos los microservicios (User, Product, Order) mediante servicio interno Kubernetes.
  • Gestión de Identidad Nombres de pods predecibles (postgres-0) esenciales para operaciones de mantenimiento y backup.
# postgres-statefulset.yaml kind: StatefulSet metadata:   name: postgres spec:   serviceName: "postgres"   replicas: 1   selector:     matchLabels:       app: postgres

Persistent Volumes (PVC)

  • Almacenamiento GKE SSD Uso de StorageClass: standard-rwo mapeado a Google Persistent Disks (SSD) para alto rendimiento I/O.
  • Aprovisionamiento Dinámico Solicitud automática de volumen (PVC) de 5Gi por réplica, garantizando persistencia tras reinicios.
  • Access Modes Configuración ReadWriteOnce para asegurar que el volumen sea montado exclusivamente por el pod de base de datos.
# Volume Claim Template volumeClaimTemplates:   - metadata:       name: postgres-pvc     spec:       accessModes: [ "ReadWriteOnce" ]       storageClassName: "standard-rwo"       resources:         requests:           storage: 5Gi
