Gestión de Secretos con Sealed Secrets

Implementación GitOps-safe para credenciales mediante criptografía asimétrica y Sealed Secrets Controller

Cifrado Asimétrico

El desarrollador cifra el secreto localmente usando la clave pública del cluster. Solo el controlador dentro del cluster (con la clave privada) puede descifrarlo.

Seguro para Repositorios

Los manifiestos SealedSecret pueden comitearse en repositorios públicos (GitHub) sin riesgo, ya que el contenido sensible es ilegible fuera del cluster.

Rotación Automática

El controlador rota las claves de cifrado cada 30 días por defecto. Los secretos antiguos siguen siendo descifrables (soporte multi-clave).

Flujo de Creación:

1 kubectl create secret ... --dry-run=client -o yaml
2 kubeseal --format=yaml > sealed-secret.yaml
3 git push (Commit seguro del archivo cifrado)
4 kubectl apply (Controller descifra y crea Secret nativo)
secret-plano.yaml (NO SEGURO)
Riesgo Alto
apiVersion: v1
kind: Secret
metadata:
  name: db-credentials
data:
  # Base64 encoded (fácilmente reversible)
  password: c3VwZXJfc2VjcmV0X3Bhc3N3b3Jk
sealed-secret.yaml (SEGURO)
GitOps Ready
apiVersion: bitnami.com/v1alpha1
kind: SealedSecret
metadata:
  name: db-credentials
  namespace: dev
spec:
  encryptedData:
    # Cifrado asimétrico (ilegitible sin clave privada)
    password: AgBy3/4...[BLOB CIFRADO NO LEGIBLE]...Xy9z
  template:
    metadata:
      name: db-credentials
Controller Status: Running (namespace: sealed-secrets)
Key Rotation: Automatic (30 days)
