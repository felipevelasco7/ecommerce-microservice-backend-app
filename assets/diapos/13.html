Persistencia, Backups y Migraciones

Estrategias de protección de datos, recuperación ante desastres y gestión del ciclo de vida de bases de datos

Backups y Recuperación

  • Backups Automatizados CronJob Jobs programados en Kubernetes para realizar volcados completos de PostgreSQL periódicamente.
  • Estrategia de Restauración Validada Procedimientos documentados y probados para restaurar datos desde volúmenes persistentes o snapshots.
  • Gestión de Datos Persistentes Protección contra eliminación accidental mediante ReclaimPolicy: Retain en volúmenes críticos.
# CronJob de Backup Automático apiVersion: batch/v1 kind: CronJob metadata:   name: postgres-backup spec:   schedule: "0 2 * * *" # Diario a las 2 AM   jobTemplate:     spec:       template:         spec:           containers:           - name: backup             command: ["/scripts/backup.sh"]

Migraciones y Ciclo de Vida

  • Migraciones de Esquema Scripts Automatización de cambios en estructura de BD mediante scripts centralizados (/scripts/scr.sh).
  • Gestión de Estado (Probes) Configuración precisa de readinessProbe y livenessProbe para asegurar integridad de datos durante arranques.
  • Graceful Shutdown Manejo correcto de señales SIGTERM para cerrar conexiones y transacciones limpiamente antes de terminar pods.
  • Inicialización de Datos Scripts de seed data para entornos de desarrollo y staging (db.sh) facilitando pruebas rápidas.
// Script de Gestión de BD #!/bin/bash function migrate_schema() {   echo "Ejecutando migraciones para $ENV..."   kubectl exec -it postgres-0 -- psql -U postgres -f /scripts/V1__init.sql }
