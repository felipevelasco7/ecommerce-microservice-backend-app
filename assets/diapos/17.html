Autoscaling Inteligente: HPA + KEDA

Escalado horizontal basado en recursos y eventos para elasticidad total

Horizontal Pod Autoscaler (HPA)

  • Escalado por Recursos Standard Configurado para todos los microservicios basado en consumo de CPU (target: 70%) y Memoria (target: 80%).
  • Límites y Requests Optimizados Definición precisa de requests/limits para evitar OOMKilled y asegurar QoS adecuado.
  • Rangos de Replicas Dinámicos Configuración min/max replicas (ej. 2-10) adaptada a la criticidad de cada servicio.
# HPA Configuration Example metrics:   - type: Resource     resource:       name: cpu       target:         type: Utilization         averageUtilization: 70

KEDA Event-Driven Autoscaling

  • 5 ScaledObjects Activos Advanced Implementados para: API Gateway, User, Product, Order y Payment Service.
  • Métricas Customizadas Escalado basado en HTTP Request Rate (Prometheus), tamaño de colas y latencia de respuesta.
  • Validación en Pruebas de Carga Capacidad demostrada de escalar de 2 a 10 pods en <1min durante picos de tráfico simulados.
  • Escalado a Cero (Scale-to-Zero) Posibilidad de apagar servicios no críticos (worker nodes) cuando no hay actividad.
# KEDA ScaledObject for API Gateway triggers:   - type: prometheus     metadata:       serverAddress: http://prometheus-server       metricName: http_requests_total       threshold: '100'       query: sum(rate(http_requests_total[2m]))
