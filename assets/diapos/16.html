Distributed Tracing con Zipkin

Rastreo distribuido de peticiones para análisis de latencia y dependencias

Integración y Propagación

  • Servicio Zipkin Centralizado Port 9411 Servidor Zipkin desplegado como microservicio independiente que recolecta spans de todo el cluster.
  • Propagación de Trace IDs Inyección automática de headers B3 (X-B3-TraceId, X-B3-SpanId) desde el API Gateway hacia los microservicios.
  • Integración Spring Cloud Sleuth Instrumentación automática en aplicaciones Spring Boot para capturar tiempos de entrada, salida y llamadas a DB.

Análisis de Dependencias y Latencia

  • Mapeo de Dependencias Visual Generación automática de grafos de dependencia entre servicios basados en el tráfico real capturado.
  • Análisis de Latencia por Hop Desglose preciso del tiempo gastado en cada salto de red y procesamiento interno de servicio.
  • Identificación de Cuellos de Botella Detección visual de servicios lentos o queries de base de datos ineficientes que afectan el tiempo total.
  • Correlación con Logs Critical Trace ID incluido en logs de aplicación (Logback + Loki) permitiendo saltar del error al trace completo.
Valor para el Negocio

Permite reducir el MTTR (Mean Time To Resolution) al identificar rápidamente qué microservicio está causando lentitud en una transacción compleja de e-commerce.
